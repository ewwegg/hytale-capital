# Code Conventions

## Purpose

Define consistent code organization patterns, import ordering to maintain readability and reduce cognitive overhead when navigating the codebase. Standardized code style enables faster code review, easier onboarding, and reduces decision fatigue during development.

## When to Use

- Creating new component files in the project
- Reviewing pull requests for code organization standards
- Refactoring existing components to align with project standards
